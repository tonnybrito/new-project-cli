What is this tool? (Simplified)

Imagine you want to build a customized car. Instead of having to manufacture every single screw, the engine, and the wheels from scratch, you use a factory machine that instantly gives you the car's frame, wheels, and a working engine.

new-project-cli is that "factory machine" for software. It creates the foundation of your project (folders, basic files, and initial settings) so you can focus only on what makes your project unique.

  • Saves Time: What takes 30 minutes to set up manually happens in seconds.
  • Ready to Build: It even installs the "engines" (dependencies) for you automatically.
  • Remote Control (Item 3): It gives you simple "Start", "Stop", and "Clean" buttons for your app, so you don't have to worry about the technical details of the server.
  • Recycle Bin (Item 4): A safe way to delete the project folder if you change your mind. It cleans up everything for you.

💡 Summary: Tool vs. Project

Don't confuse the two! new-project-cli is your factory (the tool). Once installed globally, you don't need to stay inside this folder.

You can generate your final projects anywhere on your computer. Just like you install a word processor once and then create many documents in different folders, you install this tool once and generate many projects!

Architecture Overview

A cross-platform binary tool for automated project scaffolding. Built with Node.js, supporting native elevation for administrative tasks.

  • Native Elevation: Automatic sudo or UAC (PowerShell) requests.
  • Custom Scoping: Support for any target directory via --projects-folder.
  • Multi-Runtime: Detects and executes .sh or .bat based on host environment.

1. Global Installation

Run the internal install optimizer to handle global path linking and permissions.

# Using the direct node execution (before global install)
node bin/index.js install

# OR using standard npm
npm install -g .

macOS/Linux Will prompt for sudo if required.

Windows Will trigger a UAC prompt via PowerShell.

2. Initializing Projects

Scaffold a new project environment. Values in quotes are examples and should be replaced with your desired naming.

# Default (Current Working Directory)
newproj init "my-project-name"

# Custom Path (relative or absolute)
newproj init "my-project-name" --projects-folder ~/"your-projects-folder"

Note: The --projects-folder flag handles ~/ for home directory expansion.

3. Lifecycle Commands

Execute predefined lifecycle scripts within the project directory.

# Run startApp.sh / startApp.bat
newproj start

# Run stopApp.sh / stopApp.bat
newproj stop

# Run clearCache.sh / clearCache.bat
newproj clear

# Run destroyProject.sh / destroyProject.bat
newproj destroy
  • start: "Turns on the engine" - launches the dev server and opens your app.
  • stop: "Turns off the engine" - stops all processes and frees up memory.
  • clear: "Cleans the house" - removes temporary files and corrupted caches.
  • destroy: "The Recycle Bin" - safely removes the project folder and cleans up associated files.

Visão Geral da Arquitetura

Ferramenta binária multiplataforma para scaffolding automatizado. Construída em Node.js, com suporte a elevação nativa para tarefas administrativas.

  • Elevação Nativa: Solicitações automáticas de sudo ou UAC (PowerShell).
  • Escopo Personalizado: Suporte a qualquer diretório via --projects-folder.
  • Multi-Runtime: Detecta e executa .sh ou .bat conforme o ambiente.

1. Instalação Global

Execute o otimizador de instalação para gerenciar links globais e permissões.

# Execução direta (antes da instalação global)
node bin/index.js install

# OU via npm padrão
npm install -g .

macOS/Linux Solicitará sudo se necessário.

Windows Acionará o UAC via PowerShell.

2. Inicialização de Projetos

Crie o ambiente de um novo projeto. Os valores entre aspas são exemplos e devem ser substituídos pelo nome e caminho desejados.

# Padrão (Diretório Atual)
newproj init "meu-novo-projeto"

# Caminho Personalizado (relativo ou absoluto)
newproj init "meu-novo-projeto" --projects-folder ~/"seu-diretorio-de-projetos"

Nota: A flag --projects-folder suporta ~/ para expansão da pasta do usuário.

3. Comandos de Ciclo de Vida (Lifecycle)

Execute scripts predefinidos dentro do diretório do projeto. A ferramenta detecta automaticamente se você está no Windows ou Mac/Linux.

# Inicia o servidor de desenvolvimento
newproj start

# Encerra processos e libera portas/memória
newproj stop

# Remove arquivos temporários e caches corrompidos
newproj clear

# Exclui o projeto permanentemente (pede confirmação)
newproj destroy
  • newproj start (Ligar): "Gira a chave do seu app" - liga o servidor e abre o site no seu navegador.
  • newproj stop (O Botão de Desligar): Ele encerra tudo o que estiver rodando. Use isso quando terminar o dia para "apagar as luzes" e economizar recursos do seu computador.
  • newproj clear (O Botão de Faxina): Às vezes, o computador guarda arquivos temporários (lixo) que podem travar as coisas. Esse comando faz uma limpeza completa para que tudo volte a funcionar perfeitamente.
  • newproj destroy (A Lixeira): Se você desistir do projeto, esse comando apaga a pasta inteira com segurança. É o "botão de descarte" para manter seu computador organizado.

Nota: Estes comandos buscam pelos arquivos startApp, stopApp, clearCache e deleteProject (.sh ou .bat) na pasta atual.

¿Qué es esta herramienta? (Simplificado)

Imagine que desea construir un automóvil personalizado. En lugar de fabricar cada tornillo, el motor y las ruedas desde cero, utiliza una máquina de fábrica que le entrega instantáneamente el chasis, las ruedas y un motor en funcionamiento.

new-project-cli es esa "máquina de fábrica" para software. Crea la base de su proyecto (carpetas, archivos básicos y configuraciones iniciales) para que pueda enfocarse solo en lo que hace que su proyecto sea único.

  • Ahorra Tiempo: Lo que toma 30 minutes manualmente sucede en segundos.
  • Listo para Construir: Incluso instala los "motores" (dependencias) de forma automática.
  • Control Remoto (Item 3): Le brinda botones simples de "Iniciar", "Detener" y "Limpiar" para su aplicación.
  • Papelera (Item 4): Una forma segura de eliminar la carpeta del proyecto si cambia de opinión.

💡 Resumen: Herramienta vs. Proyecto

¡No confunda los dos! new-project-cli es su fábrica (la herramienta). Una vez instalada globalmente, no necesita quedarse dentro de esta carpeta.

Puede generar sus proyectos finales en cualquier lugar de su computadora.

  • start: "Enciende el motor" - inicia el servidor y abre tu app.
  • stop: "Apaga el motor" - detiene los procesos y libera memoria.
  • clear: "Limpia la casa" - elimina archivos temporales y cachés.
  • destroy: "La Papelera" - elimina la carpeta del proyecto de forma permanente.

Note: These commands look for startApp, stopApp, clearCache, and deleteProject (.sh or .bat) files in the current folder.
